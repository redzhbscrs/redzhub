## 🔧 Troubleshooting & Common Issues

Even the best software occasionally runs into hiccups. Here are solutions to the most frequently reported problems.

### ❌ Hub Does Not Launch on Windows

**Symptoms:** Double-clicking the launcher produces no visible response.
**Solution:** Right-click the launcher and select **Run as Administrator**. If that does not help, check that your antivirus has not quarantined the file. Add an exception for the hub's installation directory.

```bash
# Check if Windows Defender is blocking the file
powershell -Command "Get-MpThreat | Where-Object {$_.Resources -like '*redzhub*'}"
```

### ❌ Modules Fail to Load

**Symptoms:** The dashboard opens but individual modules show error icons.
**Solution:** Clear the module cache and force a fresh download.

```bash
# Clear module cache (Linux/macOS)
rm -rf ~/.config/redzhub/cache/
# Restart the hub to trigger a fresh download
```

### ❌ High CPU Usage

**Symptoms:** Device becomes sluggish after activating multiple modules.
**Solution:** Switch to lite performance mode in your configuration file.

```bash
sed -i 's/performance_mode: full/performance_mode: lite/' ~/.config/redzhub/config.yml
```

### ❌ iOS Certificate Expired

**Symptoms:** The app icon turns gray and refuses to open.
**Solution:** Revisit the official download page in Safari and re-install the configuration profile. Your settings and preferences will be preserved.

### ❌ Android Overlay Not Appearing

**Symptoms:** Modules activate but no visual overlay is visible.
**Solution:** Go to **Settings → Apps → RedZ Hub → Permissions** and ensure **Display Over Other Apps** is enabled. On some devices, battery optimization may also interfere — disable it for the hub.

### ❌ Cloud Sync Not Working

**Symptoms:** Configurations do not appear on your other devices.
**Solution:** Ensure all devices are signed into the same account within the hub. Check that your internet connection is stable and that no firewall is blocking outbound connections on port 443.
